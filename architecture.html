System Architecture

Polyglot persistence, event-driven microservices, and a knowledge graph at the center of everything. 12 services, 5 database engines, one unified reasoning substrate.

12 Microservices 5 Storage Engines Event-Driven Multi-Tenant

Design Principles

Five architectural axioms

Non-negotiable constraints that shape every technical decision in the platform.

📚

Ontology Is the API

Application code never queries databases directly. Everything flows through the semantic layer, which routes to the right storage engine and applies access control.

⚡

Events Are Truth

The event log (Kafka + XTDB) is authoritative. Current state is a materialized view. Any state can be reconstructed from event history.

🔒

Security Is Structural

Access control is evaluated on every graph traversal, not just at the API boundary. ABAC + ReBAC enforced at every layer.

🧠

Intelligence Is Layered

Deterministic algorithms (constraints, optimization) run first. ML and LLM reasoning augment, never replace, the algorithmic foundation.

🤝

Human Authority Is Absolute

The system proposes; humans decide. No action affecting a student executes without explicit human approval through a governed workflow.

Polyglot Persistence

Five storage engines, one semantic layer

Each engine selected for the query patterns it handles best. The ontology layer unifies them into a single coherent API.

📈

Neo4j Enterprise 5.x

Primary graph store. Index-free adjacency delivers 10–50ms for 6+ hop traversals. Composite Databases enable per-district physical sharding. 65+ GDS algorithms built in.

🗃

PostgreSQL 16.x

Transactional OLTP for user accounts, configuration, and constraint registry. ACID guarantees with Row-Level Security for tenant isolation.

🕒

XTDB v2

Native bitemporality — valid-time + system-time on all tables. SQL:2011 superset with Postgres wire protocol. Point-in-time graph reconstruction for auditing.

🔍

Elasticsearch 8.x

Sub-50ms full-text search with faceted navigation, autocomplete, and nested object queries. Index-per-tenant isolation for secure multi-tenancy.

⚡

Redis 7.x

Sub-1ms cache reads for query results, session state, and rate limiting. Pub/Sub for event-driven cache invalidation across services.

Service Decomposition

Twelve microservices in four tiers

Built in dependency order. Each tier relies only on services from equal or lower tiers.

Tier 1 — Foundation Services (build first)

1. Ontology Metadata Service

Schema registry, entity type definitions, version control. Powers schema evolution without downtime. PostgreSQL + Redis backed.

2. Auth & Security Service

Authentication (Keycloak/Auth0), authorization (OPA Rego policies), tenant management. Evaluates ABAC + ReBAC on every request.

3. Object Storage Service

CRUD operations on ontology objects, graph writes, relationship management. The primary interface to Neo4j for all application code.

Tier 2 — Data Services (build second)

4. Ingestion Service

Source system connectors (Clever, PowerSchool, SEIS, MAP, Star), CDC consumers via Debezium, batch ETL. Raw data landing zone in S3.

5. Entity Resolution Service

Splink probabilistic matching pipeline, golden record construction, crossref management. Resolves “Johnathan Smith” across 5 systems.

6. Search Service

Full-text indexing, faceted filtering, autocomplete. Elasticsearch with index-per-tenant isolation.

Tier 3 — Intelligence Services (build third)

7. Signal Detection Service

Feature engineering (48+ features per learner), threshold/trend/anomaly/convergent/GNN detection, LightGBM risk scoring.

8. Reasoning Service

LLM agent orchestration via LangGraph, Think-on-Graph traversal, Graph-RAG, Graph-Constrained Reasoning, explanation generation.

9. Constraint & Optimization Service

Z3 real-time validation (<100ms), OR-Tools CP-SAT scheduling, Timefold assignment, Gurobi MILP. 12 optimization models.

Tier 4 — Operational Services (build fourth)

10. Case & Workflow Service

Case state machine (12 states), task management, Playbook execution engine. The operational backbone for student support cases.

11. Monitoring & Adaptation Service

Bayesian growth projections, CUSUM divergence detection, adaptation triggers, Impact Ledger. The closed-loop intelligence layer.

12. Discovery Service

Leiden community detection, GraMi subgraph mining, PrefixSpan temporal patterns, causal analysis (DoWhy + EconML).

Tenant Isolation

Multi-tenancy that never leaks

Every entity, event, query, log entry, and cache key carries a tenant ID. The Query Service injects tenant filters at the query planning layer.

EngineIsolationMechanism
Neo4jPhysicalComposite Database with per-tenant constituent databases
PostgreSQLLogicalRow-Level Security with tenant_id policies
KafkaLogicalTenant-prefixed topics (org_abc.learner.assessment)
ElasticsearchPhysicalIndex-per-tenant with filtered aliases
RedisLogicalKey prefixing (org_abc:cache:learner:456)
XTDBLogicalTenant column on all tables with query-time filtering
Tenant Identifier Format: org_{uuid} — globally unique, opaque, not derived from district name. Application code never constructs unfiltered queries.

Data Flow

Three paths through the system

Every request follows one of three well-defined paths through the architecture.

Write Path — Data In

Source System → Connector → Raw Landing (S3) → Processing Pipeline → Parse & Validate → Normalize → Enrich → Entity Resolution → Graph Write

Every write produces: a domain event on Kafka, a bitemporal record in XTDB, an audit log entry, and a search index update in Elasticsearch.

Read Path — Data Out

Client Request → API Gateway → Auth Check (OPA) → Query Service → Route to Store → Field-Level Access Control → Response

The Query Service routes to Neo4j for graph traversals, Elasticsearch for search, XTDB for temporal queries, and Redis for cached results — all transparently.

Intelligence Path — Reasoning

Signal Detected → Case Created → LangGraph Agent → Graph Traversal via Tools → Constraint Check (Z3) → Optimization Solve → Proposal Generation → Human Review → Action Plan → CUSUM Monitoring

The agent operates within strict governance: whitelisted tools, token budgets, hallucination checks, and mandatory human approval at the decision point.
