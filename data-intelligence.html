Data Pipeline

Five stages from chaos to clarity

Raw data arrives in wildly inconsistent formats. The pipeline transforms it into clean, normalized, interconnected ontology objects.

Source Systems

Connectors for every school system

🔗

Clever (Priority 1)

REST API v3.1 with OAuth 2.0. Covers 95K+ schools. Students, teachers, sections, courses, schools. Events API for change detection.

🏫

PowerSchool (Priority 2)

Plugin-based OAuth 2.0. PowerQuery for bulk extraction. CDC via Debezium for real-time change capture from on-premise databases.

📋

SEIS / Frontline (Priority 3)

CSV/SFTP export. The most valuable and most difficult integration — contains IEP goals, service minutes, provider assignments, progress notes.

📊

NWEA MAP Growth

CSV via Data Export Scheduler. RIT scores, percentiles, growth projections, subdomain breakdowns across up to 8 goal areas.

⭐

Renaissance Star

CSV extract. Scaled scores, percentile ranks, Student Growth Percentiles, zone of proximal development, instructional reading levels.

📖

Acadience (DIBELS)

CSV from Acadience Data Management. Composite scores, risk levels, subtests varying by grade (ORF, NWF, PSF, Letter Naming).

Entity Resolution

One student, five systems, one record

When “Johnathan Smith” in the SIS is “John Smith Jr.” in SEIS and “Johnathan A. Smith” in MAP — Splink resolves them probabilistically.

Before Resolution

  • Student appears in 5+ systems with inconsistent IDs
  • Name spellings differ across systems
  • Assessment data disconnected from IEP data
  • No cross-system picture possible

After Resolution

  • Golden record with canonical identity
  • Cross-reference table linking all source IDs
  • Assessment + IEP + attendance unified
  • Full-picture reasoning enabled
Splink Configuration Details

Blocking Strategy (reduce 2.5B comparisons to manageable candidates)

Rule 1: Exact DOB + first 3 chars of last name

Rule 2: Exact last name + first 3 chars of first name + same school

Rule 3: Exact district student ID (deterministic)

Rule 4: Exact state student ID (deterministic)

Rule 5: Soundex of last name + DOB year + gender (phonetic matching)

Match Thresholds

≥0.95: Auto-link (no human review). 0.70–0.95: Human review queue. <0.70: Create as new entity.

Survivorship Rules

SIS is authoritative for identity and enrollment. SPED system is authoritative for IEP/services. Assessment platform is authoritative for scores. Program flags use union strategy across all sources.

Feature Engineering

48+ features per learner, nightly

📚

Academic (14 features)

GPA trend, courses failing, grade-vs-assessment discrepancy, growth rate, standards mastery, missing assignment acceleration.

📅

Attendance (12 features)

Attendance rate (YTD, 30d, 14d), trend slope, consecutive absences, day-of-week patterns, period patterns, absence-after-event correlation.

📣

Behavioral (7 features)

Incident counts (30d, 90d), trend direction, severity escalation flag, new behavior type flag, positive behavior trend.

💻

Engagement (4 features)

LMS login frequency, login trend, assignment submission timeliness, extracurricular count changes.

🎯

Intervention (5 features)

Active count, dosage compliance, session attendance, response rate, prior intervention success rate.

📋

Compliance (6+ features)

Active IEP/504, annual review days until, service minutes compliance %, overdue evaluations, overdue meetings.

Bias Mitigation: Race, ethnicity, gender, and disability status are NEVER included as model features. The risk model predicts from behavioral and academic signals, not from identity. Post-hoc fairness audits verify calibration across demographic groups.

Signal Detection

Five methods, zero blind spots

⚠️

1. Threshold

Deterministic rules that fire when values cross boundaries. Fastest and most explainable. “Attendance below 90%”, “Failing 2+ courses.”

📈

2. Trend

OLS regression detects statistically significant slope changes. Catches declining trajectories even when no threshold is crossed. Tests prior vs. recent window slopes.

🔭

3. Anomaly

Isolation Forest detects data points inconsistent with the learner’s own history and peer group. Identifies which specific features are driving the anomaly.

🌎

4. Convergent

The most powerful type. Detects when multiple weak signals from different domains converge on the same learner within a 14-day window. A tardy + grade drop + missed session = action.

🧠

5. GNN-Based

Graph Neural Networks trained on the student similarity graph detect community-level patterns invisible to individual-level analysis.

Risk Prediction

LightGBM composite risk scoring

Predicts probability of negative outcomes to prioritize which students the system analyzes first. Never used to make decisions — only to set analysis priority.
