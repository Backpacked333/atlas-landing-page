The Problem

Schools drown in data.
Students fall through the cracks.

A typical school district runs 47 disconnected software systems. When data lives in silos, the students who need help the most are the hardest to find.

Without ATLAS

47 disconnected systems

SIS, LMS, SPED, assessments, behavior, transportation — all separate databases with no shared identity.

Manual cross-referencing

A counselor checks 6 systems to answer “Is this student at risk?” — if they think to ask at all.

Months of delayed detection

An intervention isn’t working, but nobody notices until the quarterly review because no system connects the dots.

Invisible causation

A bus route change causes late arrivals to math — fractions scores decline. No system connects cause to effect.

With ATLAS

One unified knowledge graph

Every student, teacher, course, assessment, IEP, and schedule connected in a single semantic model.

Automatic signal detection

48+ features across 5 detection methods spot at-risk students before humans could — in real time.

Intelligent recommendations

LangGraph reasons over the knowledge graph, matching students to optimal interventions and schedules.

Causal understanding

How It Works

Five steps from raw data to real action.

ATLAS transforms disconnected school data into a unified intelligence layer through a five-stage pipeline — each stage adding more context and understanding.

1

Ingest

Connect to any source system — SIS, LMS, SPED, assessments — via REST, SFTP, CSV, or CDC streaming. Raw data lands in Kafka.

Kafka · Debezium · CDC
2

Parse & Normalize

Validate schemas, deduplicate records, and normalize naming conventions. Every field maps to the ATLAS ontology.

Schema Registry · Avro
3

Resolve Entities

Probabilistic matching links “Maria Garcia” in the SIS to “M. Garcia” in SPED to student #4821 everywhere.

SpLink · DuckDB
4

Build the Graph

Resolved entities and relationships are written into a Neo4j knowledge graph with 200+ entity types across 9 domains.

Neo4j · XTDB · Postgres
5

Reason & Act

Signal detection, LangGraph reasoning, and 12 optimization models generate recommendations — humans approve every action.

LangGraph · OR-Tools · Z3

Core Principles

Designed around
five truths.

The Ontology is the API

No direct database queries. Application code interacts with semantic entities and relationships. The ontology enforces consistency across all 5 storage engines.

Events Are Truth

Current state is a materialized view of the event stream. Every change is captured, ordered, and replayable — providing complete auditability and temporal reconstruction.

Security Is Structural

Access control on every graph traversal, not just at the API boundary. ABAC + ReBAC policies evaluated on each query. FERPA and COPPA compliance by architecture.

Intelligence Is Layered

Deterministic algorithms first. ML augments. LLMs orchestrate. Each layer adds sophistication while the previous layer provides safety bounds and explainability.

Human Authority Is Absolute

The system proposes, humans decide. Every recommendation passes through an approval gate. Educators maintain full control over student-facing actions. The AI is a tool — never an authority.
