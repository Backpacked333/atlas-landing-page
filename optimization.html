Intelligence & Optimization

LangGraph agents meet mathematical solvers

LangGraph-orchestrated LLM agents reason over the knowledge graph while 12 mathematical optimization models solve scheduling, routing, and resource-matching problems at district scale.

Governed tool registry — 8 tools, zero free-text SQL

Every tool the agent can invoke is whitelisted, schema-validated, and rate-limited. The LLM never writes raw queries — it selects parameterized operations from a governed registry.

🔍 graph_query

Parameterized Cypher templates for traversing the knowledge graph. Supports learner context, cohort analysis, and relationship discovery.

📈 time_series_fetch

Retrieves temporal feature vectors from XTDB v2 bitemporality. Supports sliding windows, YoY comparison, and trend extraction.

⚠️ signal_lookup

Fetches active signals for a learner or cohort, including severity, detection method, contributing features, and trend trajectory.

🛠 intervention_catalog

Searches the intervention registry by signal type, tier, grade band, and resource requirements. Returns evidence levels and historical eROI.

💪 constraint_check

Validates a proposed action against policy constraints (class sizes, certification requirements, budget caps, scheduling conflicts).

🎯 optimization_solve

Dispatches a problem to the appropriate solver (CP-SAT, Z3, Timefold, or Gurobi) and returns the optimal or near-optimal solution.

📊 eroi_calculate

Computes the expected Return on Investment for a proposed intervention using the multi-factor eROI model with confidence intervals.

📝 explain_generate

Produces natural-language explanations with citations to data points, graph paths, and confidence scores for decision provenance.

Reasoning patterns — two core system prompts

Specialized system prompts guide the LLM through structured reasoning chains, ensuring consistent and auditable analysis.

Root-Cause Analysis Prompt

Step 1: Signal Decomposition

Break down the composite signal into contributing features with attribution weights.

Step 2: Temporal Analysis

Examine feature trajectories over time. Identify inflection points and rate-of-change anomalies.

Step 3: Graph Context

Traverse the knowledge graph to find related entities, peer comparisons, and environmental factors.

Step 4: Hypothesis Formation

Generate ranked hypotheses with confidence scores, citing specific data points as evidence.

Step 5: Intervention Matching

Map each hypothesis to evidence-based interventions from the catalog, computing eROI for each.

Intervention Matching Prompt

Constraint Awareness

Check staff availability, budget remaining, caseload limits, and scheduling conflicts before recommending.

Evidence Hierarchy

Prefer interventions with Tier 1 (strong) or Tier 2 (moderate) evidence from What Works Clearinghouse.

Equity Check

Verify proportional representation across demographic groups. Flag disparate impact exceeding threshold.

eROI Ranking

Rank all viable interventions by expected ROI, presenting top 3 with confidence intervals and trade-offs.

12 optimization models — 4 solver engines

From master bell scheduling to bus routing and staff assignment, each model is formulated as a constrained optimization problem and dispatched to the appropriate solver.

Solver Selection Logic: The system automatically dispatches each problem to the optimal solver based on problem structure. CP-SAT handles discrete feasibility, Gurobi tackles large-scale linear/integer programs, Timefold manages dynamic rescheduling, and Z3 verifies logical satisfiability for policy constraints.
Solver deep dives — CP-SAT, Gurobi, Timefold, Z3

Each solver is containerized with standardized input/output schemas. The dispatch layer handles serialization, timeout management, and solution caching.

OR-Tools CP-SAT

Google’s constraint programming solver — the workhorse for ATLAS scheduling. Handles 5 of the 12 models. Supports Boolean, integer, and interval variables with global constraints. Parallel search with portfolio of strategies. Typical solve time: <30s for a 2,000-student school’s master schedule. Warm-start from previous solutions for incremental re-optimization.

Gurobi Optimizer

Commercial-grade MILP solver for large-scale linear and integer programming. Budget allocation with 100+ constraint rows per district. Multi-year capital planning with stochastic demand scenarios. Bus routing sub-problems when OR-Tools hits scalability limits. Cloud-hosted with per-district license isolation.

Timefold (OptaPlanner)

Java-based constraint solver optimized for dynamic planning with frequent re-scheduling. Substitute teacher assignment with real-time availability updates. Late-acceptance and tabu search metaheuristics. Continuous planning mode — re-optimizes as constraints change.

Z3 SMT Solver

Microsoft’s Satisfiability Modulo Theories solver for logical constraint verification. Enrollment projection feasibility under policy change scenarios. Validates that proposed schedules satisfy all hard constraints before deployment. Unsat core extraction pinpoints the minimal set of conflicting constraints.

The eROI engine — expected return on investment

Every recommendation carries an expected ROI score computed from historical outcomes, implementation costs, and opportunity costs. This moves education decisions from intuition to evidence.

Portfolio Optimization

Given a fixed budget, the system selects the portfolio of interventions that maximizes total eROI across all flagged learners, subject to staff and scheduling constraints.

Diminishing Returns

The model accounts for diminishing returns — adding a 3rd intervention for a student may yield less marginal eROI than a 1st intervention for an unserved student.
